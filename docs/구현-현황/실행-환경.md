# 실행 환경 / 로컬 개발 설정

> 코드 기준일: 2026-07-05 · 대상: 로컬 개발(Windows)

## 로컬 실행

### 사전 조건
- **MySQL** — Windows 로컬 서비스 `MySQL96`(root/1234) 사용. `mlsoft_leave` DB(utf8mb4)가 있어야 함.
  - Docker(`backend/docker-compose.yml`)는 3306 포트를 로컬 MySQL이 이미 점유해 사용하지 않음.
- **로컬 시크릿 파일** — `backend/src/main/resources/application-local.yml` (**gitignored**). OAuth 자격증명·JWT·공휴일 API 키·DB 비밀번호·`admin-emails` 보관.

### 백엔드
```
SPRING_PROFILES_ACTIVE=local  →  ./gradlew bootRun   (http://localhost:8080)
```
- `local` 프로필이 `application-local.yml`을 덮어써 DB(root/1234)·시크릿·`allowed-domain:""`(도메인 제한 해제)를 주입.
- `application.yml`은 **시크릿을 전부 환경변수 참조**로만 둠(커밋 안전). 실제 값은 local.yml에만.

### 프론트
```
cd frontend  →  npm run dev   (http://localhost:5173)
```
- Vite proxy가 `/api` → `localhost:8080`로 전달.

## 설정 분리 원칙
| 파일 | 커밋 | 내용 |
|---|---|---|
| `application.yml` | ✅ 커밋 | 공통 설정, 시크릿은 `${ENV}` 참조만 |
| `application-local.yml` | ❌ gitignored | 실제 개발용 시크릿·자격증명 |

## 관리자 부트스트랩 (`ADMIN_EMAILS`)

개발용으로 특정 계정을 총관리자로 만드는 방법. **중요한 함정 하나가 있음:**

> `ADMIN_EMAILS`(`app.admin-emails`)는 코드상 **최초 가입(자동 회원가입) 시에만** SYSTEM_ADMIN을 부여한다 (`CustomOAuth2UserService.registerNewUser`). 이미 EMPLOYEE로 가입된 계정은 `ADMIN_EMAILS`에 추가하고 재로그인해도 **승격되지 않는다.**

따라서 이미 가입된 계정을 관리자로 만들려면 **두 가지를 함께** 해야 한다:
1. `application-local.yml`의 `app.admin-emails`에 이메일 추가 (신규 가입/재생성 대비 + 올바른 설정)
2. **기존 DB 계정의 role을 직접 승격:**
   ```sql
   UPDATE users SET role='SYSTEM_ADMIN' WHERE email='<개발용 Google 계정>';
   ```
3. 백엔드 재기동 + **재로그인**(로그아웃 → Google 로그인) — 새 JWT·`userInfo.role`을 받아야 프론트 라우트 가드/사이드바에 관리자 메뉴가 나타남.

> 현재 개발용 로그인 계정은 위 절차로 SYSTEM_ADMIN 상태. 실제 이메일 값은 `application-local.yml`의 `admin-emails` 참조.

## OAuth 주의
- Google Cloud Console "승인된 리디렉션 URI"에 `http://localhost:8080/login/oauth2/code/google`가 등록돼 있어야 로그인 완료 (없으면 `redirect_uri_mismatch`).
- 메일 계정(`MAIL_USERNAME/PASSWORD`)은 서버 기동용 더미. 실제 발송 기능 구현 시 교체 필요.

## 서버 재기동 / 종료 (개발 팁)
- 백그라운드로 띄우면 세션 종료 시 함께 죽는 현상이 있어, **세션과 분리된 독립 프로세스**로 띄우는 것을 권장(Windows `Start-Process` 등). 로그는 파일로 리다이렉트.
- 백엔드 재기동: 8080 리스닝 자바 프로세스 종료 후 `bootRun` 재실행.
- 종료: 8080(백엔드)·5173(프론트) 프로세스 각각 종료.
