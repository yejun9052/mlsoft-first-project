- `src/mocks/data.js` — 가상 회사 MLsoft(부서 3개·구성원 12명+퇴직 1·연차/복리 신청·공휴일·정책·설정·리셋 이력)의 **정합적 목업 데이터**. `getCurrentUser()`는 실제 로그인 정보(localStorage) + mock 상세를 병합.
- `src/components/ui/` — `PageHeader`, `StatusBadge`(상태 틴트 pill), `Card`(서피스 래퍼)

## ✅ 최종 리뷰 결함 수정 (2026-07-07, Fable 검증)

- **401 처리·로그아웃 시 `localStorage.clear()` → `userInfo`만 제거** — 무관한 키까지 지우지 않게 (`api/index.js`, `Sidebar`)
- **신청 패널 보강** — 창 리사이즈 시 화면 밖 고립 방지 재클램프 · Esc 닫기 · `role="dialog"` 접근성 · `onPointerCancel` 처리 · 기본/서브 승인자 **동일인 이중 지정 방지**
- **대시보드** — 빠른 신청 버튼이 toast 안내 대신 **실제 페이지로 링크**(연차·반차→캘린더, 경조사→복리후생), '내 결재 대기' 건수에 `CANCEL_PENDING` 합산
- **캘린더 React key 안정화** — index key → 날짜/식별자 key
- 이에 앞서 대시보드·캘린더·신청 패널 **타이포 확대** (가독성, 07-06)

## ❌ 미구현 / 주의
- **실 API 연동** — 9개 페이지 전부. 백엔드 업무 API 완성 후 전환 필요.
- **연차 신청 폼**(디자인 ③) — 전용 라우트·페이지는 없음. 캘린더의 날짜 클릭 → `LeaveApplyPanel`이 폼 역할을 대신함(제출 mock). 대시보드의 빠른 신청 버튼은 캘린더·복리후생 페이지로 링크됨. 전용 폼 페이지를 둘지 캘린더 방식으로 일원화할지는 **결정 대기**.
- **공휴일** — `mocks/data.js`의 하드코딩 목록. 실제로는 `/api/holidays`(data.go.kr) 연동 대상.

## 🔄 실 API 전환 방법 (백엔드 API 완성 후)
1. 각 페이지 상단 `import { ... } from '../mocks/data.js'` → `src/api/*` 호출로 교체
2. 데이터 로딩은 `useEffect` + 로딩/에러 상태 처리 (axios 인스턴스 `api` 재사용)
3. mock 액션(`toast`)을 실제 POST/PATCH 호출로 교체
4. 전환 완료 후 `src/mocks/` 삭제 (파일 상단 주석에 명시)
