### 2) 연차 코어 API (`domain/leave`) — 2026-07-10 신규
`LeaveController` (`/api/leaves`) 11개 엔드포인트 + `LeaveService` + DTO 7종 (`0cac59f`). 본인 식별은 `AuthUser`에서 추출(body의 사용자 ID 신뢰 금지), `@PreAuthorize`는 역할 게이트만 — 소유/승인자 식별 검증은 서비스 계층.

| Method | URL | 설명 | 권한 |
|---|---|---|---|
| POST | `/api/leaves` | 신청 — **선차감**, 주말·과거·기간중복·잔여 검증 | 전체 |
| GET | `/api/leaves/me` | 내 신청 내역 (페이징, status 필터) | 전체 |
| GET | `/api/leaves/me/summary` | 잔여 현황 (총·사용·잔여·대기·당겨쓰기·다음 기산일) | 전체 |
| GET | `/api/leaves/calendar` | 캘린더용 승인 연차 (year·month, 타인 사유 마스킹) | 전체 |
| GET | `/api/leaves/team` | 내 팀 연차 현황 (기간 필터, 미지정 시 이번 달) | 전체 |
| GET | `/api/leaves/pending` | 내가 승인자인 대기 목록 (취소 대기 포함) | TL·SA |
| GET | `/api/leaves` | 전체 신청 목록 (status·keyword 필터) | SA |
| POST | `/api/leaves/{id}/approval` | 승인/반려 — 조건부 갱신 | TL·SA (승인자만) |
| POST | `/api/leaves/{id}/cancel` | 취소 — 미래=즉시 취소+복구, 과거 포함=소급취소(`CANCEL_PENDING`) | 본인 |
| POST | `/api/leaves/{id}/cancel-approval` | 소급취소 승인(확정+복구)/반려(APPROVED 복원) | TL·SA (승인자만) |
| GET | `/api/leaves/{id}/histories` | 해당 건 처리 이력 | 본인·승인자·SA |

**동시성 방어 (docs/01 R-5):**
- 이중 처리 → **조건부 갱신(CAS)**: `WHERE status=기대값` UPDATE, rowcount 0이면 `ALREADY_PROCESSED`(409). `@Modifying(clearAutomatically·flushAutomatically)` 후 재조회로 부수효과(차감/복구) 적용.
- 잔여 초과 경쟁 → `User.@Version` 낙관적 락 (충돌 시 409 자동).

**구현 방식:** 엔티티 변경 없음 — 기존 `LeaveRequest` 상태전이·선차감/복구 메서드·`RequestStatus` 재사용, 마이그레이션 없음. `LeaveRequestRepository`에 조회 9 + 조건부 갱신 2 쿼리 추가, `UserRepository`에 승인자 SA fallback, `ErrorCode`에 주말·과거 400 추가. 이메일 알림은 `// TODO(email)` 시밍만 (발송 인프라 마일스톤에서), 공휴일 검증은 holidays 마일스톤으로 분리(주말만 400).

**테스트:** `LeaveServiceTest` 18건(단위) + `LeaveServiceIntegrationTest` 1건(H2 — 신청→요약→조회 6종→승인→취소 + 조건부 갱신을 실제 SQL로 검증). 전체 29/29 그린.

### 3) 시큐리티 (`security/`, `config/SecurityConfig`)
- **무상태(STATELESS) JWT 쿠키 인증** + Google OAuth2 로그인 (`SecurityConfig`)
- `CustomOAuth2UserService` — ① `email_verified` 검증 ② 이메일 도메인 검증(`app.allowed-domain` 비면 스킵) ③ 미가입 자동 가입(EMPLOYEE+미배정, `ADMIN_EMAILS`면 SYSTEM_ADMIN) ④ 퇴직자(`is_active=false`) 로그인 거부
- `JwtProvider` / `JwtFilter` (서블릿 이중 등록 차단 처리) / `TokenCookieFactory` (httpOnly·SameSite=Lax·Secure 플래그)
- `OAuth2SuccessHandler` / `OAuth2FailureHandler` — 성공 시 `/oauth-callback`, 실패 시 `/login?error=...`
- `OnboardingCheckInterceptor` — 온보딩 미완료 유저 `/api/auth/*` 외 차단
- `RestAuthenticationEntryPoint` — 미인증 접근 → 401 JSON (기본 로그인 페이지 리다이렉트 방지)
- CORS: `app.frontend-url`만 허용 + credentials

### 5) 초기 데이터 (`config/DataInitializer`)
재기동 안전(존재 시 스킵). 적재 항목:
- 근속년수별 연차 정책 21건 — `MIN(15 + (년차-1)/2, 25)` (근로기준법 §60)
- 연차 시스템 설정 3키 — `advance_leave_enabled=false`, `reminder_list_days=30`, `reminder_auto_cycle=NONE`
- 복리후생 정책 14건 (결혼·회갑·칠순·출산·졸업·조의)
- "미배정" 부서 (신규 가입 기본 소속)
