# 백엔드 구현 현황

> Spring Boot 3 · Java 21 · Spring Security · Spring Data JPA · MySQL · OAuth2 Client · jjwt
> 코드 기준일: 2026-07-08

## ✅ 구현 완료

### 1) 인증 API (`domain/auth`)
`AuthController` (`/api/auth`) — 실제 존재하는 유일한 업무 컨트롤러.

| Method | URL | 설명 | 상태 |
|---|---|---|---|
| GET | `/api/auth/me` | 내 정보 조회 (온보딩 여부 포함) | ✅ |
| POST | `/api/auth/onboarding` | 생일·입사일 입력 → base_days 정책 자동 계산 | ✅ |
| POST | `/api/auth/logout` | JWT 쿠키 즉시 만료 | ✅ |
| — | `/oauth2/authorization/google` | Google 로그인 진입 (Spring 제공) | ✅ |
| — | `/login/oauth2/code/google` | OAuth2 콜백 | ✅ |

### 2) 시큐리티 (`security/`, `config/SecurityConfig`)
- **무상태(STATELESS) JWT 쿠키 인증** + Google OAuth2 로그인 (`SecurityConfig`)
- `CustomOAuth2UserService` — ① `email_verified` 검증 ② 이메일 도메인 검증(`app.allowed-domain` 비면 스킵) ③ 미가입 자동 가입(EMPLOYEE+미배정, `ADMIN_EMAILS`면 SYSTEM_ADMIN) ④ 퇴직자(`is_active=false`) 로그인 거부
- `JwtProvider` / `JwtFilter` (서블릿 이중 등록 차단 처리) / `TokenCookieFactory` (httpOnly·SameSite=Lax·Secure 플래그)
- `OAuth2SuccessHandler` / `OAuth2FailureHandler` — 성공 시 `/oauth-callback`, 실패 시 `/login?error=...`
- `OnboardingCheckInterceptor` — 온보딩 미완료 유저 `/api/auth/*` 외 차단
- `RestAuthenticationEntryPoint` — 미인증 접근 → 401 JSON (기본 로그인 페이지 리다이렉트 방지)
- CORS: `app.frontend-url`만 허용 + credentials

### 3) 도메인 모델 — 엔티티·리포지토리 (업무 로직 API는 아직 없음)
전 도메인의 JPA 엔티티·리포지토리는 완비. **컨트롤러/서비스가 없어 아직 화면과 연결되지 않음.**

| 도메인 | 엔티티 |
|---|---|
| user | `User`, `Role`(SYSTEM_ADMIN/TEAM_LEADER/EMPLOYEE) |
| department | `Department` (leader_id 명시) |
| leave | `LeaveRequest`, `LeaveActionHistory`, `LeaveResetHistory`, `LeaveType` |
| welfare | `WelfarePolicy`, `WelfareRequest`, `WelfareActionHistory`, `WelfareTarget` |
| holiday | `Holiday` |
| policy | `LeavePolicy`, `LeavePolicyConfig` (+ `LeavePolicyService`) |
| email | `EmailHistory`, `EmailStatus`, `EmailType` |
| common | `RequestAction`(7종), `RequestStatus` |

### 4) 초기 데이터 (`config/DataInitializer`)
재기동 안전(존재 시 스킵). 적재 항목:
- 근속년수별 연차 정책 21건 — `MIN(15 + (년차-1)/2, 25)` (근로기준법 §60)
- 연차 시스템 설정 3키 — `advance_leave_enabled=false`, `reminder_list_days=30`, `reminder_auto_cycle=NONE`
- 복리후생 정책 14건 (결혼·회갑·칠순·출산·졸업·조의)
- "미배정" 부서 (신규 가입 기본 소속)

### 5) 공통 모듈 (`global/`)
`BaseTimeEntity`, `CommonResponse`/`ResponseMessage`(성공 포맷), `ErrorCode`/`BusinessException`/`GlobalExceptionHandler`(실패 포맷 + 한글 메시지), `AppProperties`(app.* 바인딩).

### 6) 최종 리뷰 결함 수정 (2026-07-07, Fable 검증)

| 파일 | 수정 내용 |
|---|---|
| `SecurityConfig` | ERROR 디스패치 permitAll (Security 6+) — 필터 계층 예외 시 401이 실제 원인을 덮지 않게 |
| `AuthService` | 연차 계산 기준일을 **KST 고정**(`LocalDate.now(KST)`) — 서버 TZ가 UTC여도 하루 오차 방지 |
| `GlobalExceptionHandler` | 파라미터 타입 불일치·필수 누락·메서드 검증 실패 → **400 명시 처리** (catch-all 500 방지, docs/03 응답 계약) |
| `OnboardingCheckInterceptor` | 퇴직자도 **로그아웃 경로만 허용** — 잔존 httpOnly 쿠키를 본인이 정리할 수단 유지 |
