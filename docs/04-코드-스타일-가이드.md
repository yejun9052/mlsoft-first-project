## 백엔드 (Java / Spring Boot)

### 엔티티
```java
@Entity
@NoArgsConstructor(access = AccessLevel.PROTECTED)  // 무분별한 기본 생성자 방지
@Getter                                             // Setter 금지
@AllArgsConstructor
@Builder
@Table(name = "users")
public class User extends BaseTimeEntity { ... }
```
- **Setter 없이 도메인 메서드로만 상태 변경**: `approve()`, `cancel()`, `resetAndApplyPolicy(...)` 처럼 상태 전환·검증을 엔티티 안에 둔다.
- **정적 팩토리 메서드** 선호: `User.create(...)`, `LeaveRequest.create(request, user)`
- `BaseTimeEntity` 상속으로 `createdAt` 자동 관리 (`@CreatedDate` + `@EnableJpaAuditing`)
- Enum은 `@Enumerated(EnumType.STRING)` — 절대 숫자 저장 금지
- **연차 등 일수 수치는 double/Double 금지 → `BigDecimal`** (DB는 DECIMAL(4,1)). 비교는 `==`가 아니라 `compareTo` (부동소수점 오차 방지, 검증 R-1)
- User 엔티티에 `@Version` 낙관적 락 (동시 신청 초과 방지, 검증 R-5)

### 컨트롤러
- 모든 응답 `ResponseEntity<CommonResponse<?>>` 래핑
- **메시지 하드코딩 금지 — 상수(변수)로 관리**. 성공 메시지는 `ResponseMessage` 상수 클래스에 모은다:
```java
// global/response/ResponseMessage.java — 성공 메시지 전용
public final class ResponseMessage {
    public static final String LEAVE_CREATED  = "연차 신청이 완료되었습니다.";
    public static final String LEAVE_APPROVED = "승인 처리가 완료되었습니다.";
    public static final String LEAVE_CANCELLED = "연차 신청이 취소되었습니다.";

    private ResponseMessage() {} // 인스턴스화 방지
}

// 사용 — 컨트롤러에서 문자열 리터럴 직접 쓰지 않기
return ResponseEntity.ok(CommonResponse.success(ResponseMessage.LEAVE_CREATED, data));
```
- 실패는 예외 던지기 (컨트롤러에서 try-catch 금지)
- `@PreAuthorize("hasRole('TEAM_LEADER') or hasRole('SYSTEM_ADMIN')")` 메서드 레벨
- 본인 식별은 `Authentication`에서 (요청 body의 employeeId 신뢰 금지)

### 예외
- `RuntimeException` 직접 사용 금지 → 커스텀 예외 (`UserNotFoundException` 등)
- **에러 메시지도 변수로 관리** — `ErrorCode` enum에 HTTP 상태 + 메시지를 함께 정의 (docs/03의 에러 코드 표와 1:1 대응):
```java
// global/exception/ErrorCode.java
@Getter
@RequiredArgsConstructor
public enum ErrorCode {
    USER_NOT_FOUND(404, "사용자를 찾을 수 없습니다."),
    ACCESS_DENIED(403, "접근 권한이 없습니다."),
    INSUFFICIENT_LEAVE_BALANCE(400, "잔여 연차가 부족합니다."),
    OVERLAPPING_LEAVE_REQUEST(409, "이미 신청된 기간과 중복됩니다."),
    ALREADY_PROCESSED(400, "이미 처리된 신청입니다.");

    private final int status;
    private final String message;
}

// 커스텀 예외는 ErrorCode를 물고 다닌다
public class BusinessException extends RuntimeException {
    private final ErrorCode errorCode;
    public BusinessException(ErrorCode errorCode) {
        super(errorCode.getMessage());
        this.errorCode = errorCode;
    }
}
```
- `GlobalExceptionHandler`(@RestControllerAdvice)에서 `ErrorCode`의 status/message로 일괄 변환 → 예외 타입마다 핸들러 만들 필요 없이 `BusinessException` 하나로 처리 가능
- 상태 코드: 404 NotFound / 409 Conflict / 400 BadRequest / 403 Forbidden

### 서비스
```java
@Slf4j
@Service
@RequiredArgsConstructor   // 생성자 주입
public class LeaveService {
    @Transactional          // 쓰기 메서드에만
    public void addLeave(...) { ... }

    private User findUserOrThrow(String email) { ... }  // 검증은 private 헬퍼로 분리
}
```
- 스케줄러: `@Scheduled(cron = "0 0 0 * * *")` + `@Transactional` 조합, `log.info()` 로깅

### Repository
- 파생 쿼리 메서드 우선 (`findByStatusOrderByIdAsc`), 복잡한 건 JPQL
- 중첩 필드는 `_` 표기: `findByUser_EmailOrderByIdDesc`

### 주석
- 공개 메서드는 JavaDoc(`/** */`), 내부 로직은 한글 `//` 주석 + 불릿 스타일
```java
/**
 * 연차 승인 또는 반려.
 * - 권한 검증: SYSTEM_ADMIN 또는 지정 승인자만 처리 가능
 * - 승인 시: status APPROVED + 사용일수 차감
 */
```

## 프론트엔드 (React)

### 컴포넌트
- `export default function ComponentName({ props })` — **화살표 함수 컴포넌트 금지**
- Props 구조분해, 상태는 `[value, setValue]` 페어 네이밍
- useEffect 안에서 async 함수 정의 후 즉시 호출 패턴:
```jsx
useEffect(() => {
  const load = async () => { ... };
  load();
}, []);
```

### API 모듈 (src/api/*.js)
- axios 인스턴스 1개 (`withCredentials: true`) + 응답 인터셉터에서 에러 toast 일괄 처리
- 함수 하나 = 엔드포인트 하나, 함수 위에 한글 주석
- 쿼리스트링은 `URLSearchParams`, 기본값은 시그니처에 (`page = 0, size = 8`)

### 상수·유틸
- 상수는 컴포넌트 파일 최상단 `UPPER_SNAKE_CASE` 객체/배열 (`LEAVE_LABEL`, `DAY_LABEL`)
- 날짜는 `YYYY-MM-DD` 통일, 방어적 변환 유틸 (`toDateStr`)

### 스타일 (변경: BEM CSS → Tailwind)
- 이전: 페이지별 CSS 파일 + BEM(`stat-card__icon--blue`) → **이번엔 Tailwind CSS**
- 디자인 토큰은 `tailwind.config.js`에 등록해서 사용 (05-디자인-가이드.md 참고)
- 반복되는 조합은 컴포넌트로 추출 (className 문자열 상수 남발 금지)
